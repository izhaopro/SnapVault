# SnapVault 系统设计文档（整合版）

---

## 1. 文档概览

### 1.1 背景

SnapVault 是一款本地文件快照工具，用于将用户选定的文件与目录按照预定义规则展开并复制到快照目录中。系统可选生成 $7z$ 压缩归档，并提供 GUI 操作界面、配置持久化、日志追踪、进度可视化与错误容错机制。全链路采用统一 UTC 时间规范，保障日志完整性，支持路径变量与通配符解析，确保交互行为可预测、执行流程可恢复。

### 1.2 目标

- 统一时间基准：执行全链路采用 UTC（覆盖文件名、日志、预览、状态）
- 配置可保存、可加载，且具备容错能力
- Source Items 支持路径变量与通配符展开
- 明确输入校验规则并联动按钮可用性
- 保证日志"本次执行全量不丢失"
- GUI 行为一致、可预测、可测试
- 执行流程完整覆盖 `CreateArchive=true/false` 两条路径

### 1.3 非目标

- 不提供具体实现代码
- 不定义跨平台实现方案（以 Windows GUI 与路径语义为准）
- 不包含远程存储、增量备份、权限托管等扩展能力

### 1.4 术语与约定

| 名称 | 定义/说明 | 示例/备注 |
|------|-----------|-----------|
| ScriptDir | 脚本所在目录（取不到则当前目录） | 例如：`D:\Tools\SnapVault` |
| ScriptDirName | `Split-Path -Leaf ScriptDir` | 作为默认快照名来源 |
| `${APPDIR}` | 可移植路径变量，运行时等价于 ScriptDir | UI 展示时以前缀 `...` 呈现 |
| Display String | UI 显示用路径字符串 | 当路径位于 ScriptDir 下时，前缀显示为 `...` |
| 时间戳基准 | 执行开始时锁定 UTC 时间，贯穿全流程 | 同一轮执行中，文件名与日志时间均从锁定值派生 |

## 2. 需求与约束

### 2.1 功能需求

1. **Source Items 管理**：手动输入、文件添加、目录添加、删除选中、清空
2. **参数输入**：SnapshotName、OutputLocation、归档选项
3. **预览能力**：显示 Local/UTC 当前时间与快照命名预览
4. **执行能力**：复制文件、写配置副本、写日志、可选归档、可选删除快照目录
5. **配置能力**：保存到固定路径、加载并容错
6. **日志能力**：UI 实时日志 + 全量权威日志缓冲
7. **进度能力**：分阶段进度映射与状态文本
8. **执行结束恢复**：解锁 UI、清理状态与缓冲

### 2.2 非功能需求

| 维度 | 要求 |
|------|------|
| 可用性 | 非法输入即时反馈；快捷键行为一致 |
| 可靠性 | 日志完整性保障（首写 + 收尾覆盖写双重机制） |
| 可维护性 | 核心函数职责清晰、命名统一采用英文 |
| 兼容性 | 配置缺失、类型错误、JSON 损坏均可容错 |
| 可观测性 | 关键节点日志化（含 WARN/ERROR 分级） |

### 2.3 规则映射

| 规则域 | 规则定义 |
|--------|----------|
| 时间格式 | 文件名时间戳：`yyyyMMddTHHmmssZ`；日志行时间戳：`yyyy-MM-ddTHH:mm:ssZ`；UI 本地时间：`yyyy-MM-ddTHH:mm:ss`；UI UTC 时间：`yyyy-MM-ddTHH:mm:ssZ` |
| 排序规则 | Source Items 先 `FOLDER`（字母升序）再 `FILE`（字母升序） |
| 变量解析 | `${APPDIR}` 在执行期解析为 ScriptDir；展示时可折叠为 `...` |
| 通配符 | 支持 `*`、`?`、可选 `**`；`OutputLocation` 禁止通配符 |
| 去重规则 | 路径大小写不敏感去重（基于完全限定路径） |
| 日志机制 | `LogMessages`（UI 消费队列）+ `AllLogEntries`（append-only 权威记录） |
| 归档条件 | 仅当 7-Zip 可执行文件可用时启用；不可用时归档控件禁用并告警 |
| 快捷键 | TextBox：Ctrl+C/V/X/A；ListBox：Ctrl+C/A/Delete |

### 2.4 冲突处理

| 冲突点 | 处理依据 | 最终结论 | 兼容/迁移方案 |
|--------|----------|----------|---------------|
| 进度分段在不同描述处起止略有差异 | 以阶段语义完整、覆盖最终日志收尾为准 | 保留两种分段定义：`CreateArchive=true` 为 $0$-$5$ / $5$-$70$ / $70$-$80$ / $80$-$95$ / $95$-$100$；`CreateArchive=false` 为 $0$-$5$ / $5$-$85$ / $85$-$95$ / $95$-$100$ | 实现中通过配置常量定义阶段区间；回归测试分别覆盖 |
| UI 坐标存在"基础位置/7z 警示位移后位置"双值 | 以 `lbl7zWarn` 存在时整体下移 $24px$ 的布局规则为准 | 坐标保留双值（原值 / 位移后值） | 渲染时根据 `lbl7zWarn.Visible` 动态应用偏移 |
| 文档中个别字符转码异常（符号、控件类型名称） | 以语义可确定项与配套规则表互证 | 对可判定语义采用规范名；对不可判定项标注"待补充+影响" | 发布前执行一次术语与控件清单核对 |

### 2.5 信息缺失（待补充项）

| 待补充项 | 影响范围 |
|----------|----------|
| 部分转码异常字段的精确字面值 | 可能影响自动化校验脚本的字符串比对 |
| MessageBox 文案全集（部分被遮蔽） | 影响国际化与 UI 一致性验收 |
| 压缩失败 exit code 全量映射 | 影响错误分级统计与用户指引精细度 |
| Copy 冲突落位策略在极端同名链路下的完整规则 | 影响复杂目录结构下行为可预期性 |
| 系统对话框与 MessageBox 的精确尺寸/屏幕坐标 | 未在代码中明确 |
| 部分文档规则与代码实现细节差异原因 | 未在文档中明确 |

## 3. 方案设计

### 3.1 系统架构

```
┌─────────────────────────────────────────────────┐
│                   UI 层                          │
│  MainForm + 控件事件 + Timer（预览/日志/完成）    │
├─────────────────────────────────────────────────┤
│                  应用层                           │
│  参数校验 │ 路径解析 │ Source 展开 │ 复制编排     │
│  归档编排 │ 状态机管理                            │
├─────────────────────────────────────────────────┤
│                  持久层                           │
│  配置读写 │ 日志写盘 │ 目录/文件操作              │
├─────────────────────────────────────────────────┤
│                外部依赖                           │
│  7-Zip 可执行探测与调用                           │
└─────────────────────────────────────────────────┘
```

### 3.2 执行流程

| 阶段 | 名称 | 说明 |
|------|------|------|
| 1 | Validate | 输入合法性校验（SourceItems、SnapshotName、OutputLocation）、按钮联动 |
| 2 | Init | 锁定 UTC 时间戳（`LockedUtcTimestamp` / `FileNameTimestamp`）、初始化运行态状态 |
| 3 | Prepare | 创建输出目录与快照目录 |
| 4 | Expand | `${APPDIR}` 变量替换、通配符解析、路径去重、统计计数 |
| 5 | Copy | 按项逐一复制，记录 SKIPPED/FAILED |
| 6 | Save | 写入配置副本（`.config.json`）+ 首次全量日志（`.log`） |
| 7 | Archive | 可选：调用 7-Zip 生成归档；可选：归档成功后删除快照目录 |
| 8 | Finalize | Drain 日志队列 + 覆盖写最终日志文件（`AllLogEntries` 全量） |
| 9 | Unlock | 恢复 UI 状态，清理运行态缓冲，弹出成功/失败消息 |

### 3.3 模块划分

| 模块 | 职责 |
|------|------|
| Config | Load / Save / Apply 配置；容错处理 |
| Validate | SnapshotName / OutputLocation / ManualInput 校验 |
| Path | Display / Portable / Resolve 路径转换（`${APPDIR}` ↔ ScriptDir ↔ `...`） |
| Expand | 通配符匹配与递归解析 |
| Copy | 文件/目录复制与冲突处理 |
| Archive | 7-Zip 探测（PATH → 默认安装路径）、命令封装、结果处理 |
| Log | Add-LogMessage、首写、收尾覆盖写 |
| UI State | Lock / Unlock、按钮状态、计数刷新、进度推进 |

### 3.4 数据模型

#### 3.4.1 配置文件与环境路径

| 名称 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `ScriptDir` | String | 脚本目录 | `${APPDIR}` 等价目标 |
| `ScriptDirName` | String | `Split-Path -Leaf ScriptDir` | 作为默认快照名来源 |
| `ConfigFileName` | String | `SnapVault.config.json` | 配置文件名 |
| `ConfigPath` | String | `ScriptDir\SnapVault.config.json` | 固定保存/加载位置 |
| `LastBrowsePath` | String | `ScriptDir` | 文件/目录浏览初始路径 |
| `7zPath` | String/null | 启动探测结果 | 探测顺序：PATH → `C:\Program Files\7-Zip\7z.exe` → `C:\Program Files (x86)\7-Zip\7z.exe` |

#### 3.4.2 业务配置项

| 配置项 | 类型 | 必填 | 默认值 | 输入范围 | 校验/限制 | 依赖关系 |
|--------|------|:----:|--------|----------|-----------|----------|
| `SourceItems` | `String[]` | 是（执行前） | 空数组 | 绝对路径或 `${APPDIR}`；支持 `*` `?` 可选 `**` | 大小写不敏感去重；无匹配 WARN；最终为空 ERROR | 影响排序、计数、展开复制 |
| `SnapshotName` | `String` | 否 | `ScriptDirName` | 普通字符串 | 禁止 `/ \ : * ? " < > \|` | 影响目录/日志/config/归档命名 |
| `OutputLocation` | `String` | 否 | `ScriptDir` | 绝对路径或 `${APPDIR}` | 禁止通配符；不可创建则 ERROR | 影响所有输出路径 |
| `CreateArchive` | `Boolean` | 否 | `true`（仅 7z 可用） | `true/false` | 7z 不可用时强制 `false` 且禁用 | 控制是否执行归档阶段 |
| `DeleteFolderAfterArchive` | `Boolean` | 否 | `false` | `true/false` | 仅归档成功后才尝试删除 | 依赖 `CreateArchive=true` |

#### 3.4.3 运行态字段

| 字段 | 类型 | 初值 | 说明 |
|------|------|------|------|
| `Progress` | $Int(0\text{-}100)$ | `0` | 当前进度百分比 |
| `StatusText` | String | `Preparing...` | 当前状态描述文本 |
| `LogMessages` | List\<String\> | 空 | UI 消费队列 |
| `AllLogEntries` | List\<String\> | 空 | append-only 权威全量日志 |
| `IsCompleted` | Boolean | `false` | 执行是否完成 |
| `IsError` | Boolean | `false` | 是否存在关键错误 |
| `ErrorMessage` | String | 空 | 错误描述 |
| `ResultFolder` | String | 空 | 快照目录路径 |
| `ResultArchive` | String | 空 | 归档文件路径 |
| `LockedUtcTimestamp` | String | 每次运行开始锁定 | 格式：`yyyy-MM-ddTHH:mm:ssZ` |
| `FileNameTimestamp` | String | 每次运行开始锁定 | 格式：`yyyyMMddTHHmmssZ` |

### 3.5 接口与输入输出约束

| 项目 | 规则 |
|------|------|
| 配置文件路径 | `ScriptDir\SnapVault.config.json`，UTF-8 编码 JSON |
| 快照目录命名 | `{SnapshotName}_{FileNameTimestamp}` |
| 日志文件命名 | 与快照同前缀 + `.log` |
| 配置副本命名 | 与快照同前缀 + `.config.json` |
| 归档文件命名 | 与快照同前缀 + `.7z` |

## 5. 交互规则与联动

### 5.1 输入校验

| 控件 | 校验规则 | 非法反馈 |
|------|----------|----------|
| `txtSnapshotName` | 禁止字符：`/ \ : * ? " < > \|` | 红底高亮 + 禁用 Create 按钮 |
| `txtOutputLocation` | 仅接受绝对路径或 `${APPDIR}` 开头路径；禁止 `* ? **` | 红底高亮 + 禁用 Create 按钮 |
| `txtManualInput` | 允许绝对路径、`${APPDIR}` 路径、`* ? **` 通配符 | 非法时禁用 Add 按钮 |

### 5.2 快捷键

| 控件类型 | 快捷键 |
|----------|--------|
| `lstSourceItems` | `Ctrl+A` 全选；`Ctrl+C` 复制选中行；`Delete` 删除选中 |
| 文本框（全部） | `Ctrl+A` 全选；`Ctrl+C/V/X` 系统常规快捷键沿用 |

### 5.3 按钮可用性联动

| 条件 | 联动效果 |
|------|----------|
| 列表为空 | `Remove Selected` / `Remove All` 不可用 |
| 有选中项 | `Remove Selected` 可用 |
| 有任意项 | `Remove All` 可用 |
| `chkCreateArchive=false` | `chkDeleteFolder` 强制取消并禁用 |
| 未找到 7-Zip | `chkCreateArchive` 与 `chkDeleteFolder` 均禁用，显示 `lbl7zWarn` |

### 5.4 执行期联动（Lock-UI）

执行期间禁用以下控件：

`btnCreateSnapshot`、`btnSaveConfig`、`btnLoadConfig`、`btnAddFile`、`btnAddFolder`、`btnRemoveSelected`、`btnRemoveAll`、`btnBrowseOutput`、`btnManualAdd`、`txtManualInput`、`lstSourceItems`、`txtSnapshotName`、`txtOutputLocation`、`chkCreateArchive`、`chkDeleteFolder`

执行完成后恢复；归档相关控件恢复时仍受 7-Zip 可用性约束。

## 6. 实现细节

### 6.1 配置策略

- **Save**：始终写入 `ScriptDir\SnapVault.config.json`；ScriptDir 下路径自动转换为 `${APPDIR}` 形式
- **Load**：
  - 文件缺失 → 使用默认值
  - 字段类型错误 → 忽略该字段，使用默认值
  - JSON 解析失败 → 记录 WARN 并全部使用默认值

### 6.2 异常降级策略

| 异常场景 | 处理方式 | 日志级别 |
|----------|----------|----------|
| 通配符无匹配 | 记录并继续 | WARN |
| 文件占用/权限拒绝 | 标记 SKIPPED，继续 | WARN |
| 复制 I/O 错误 | 标记 FAILED，继续并汇总 | ERROR |
| 7-Zip 不可用 | 禁用归档相关控件 | WARN |
| 压缩执行失败 | 保留快照目录并记录 | ERROR |
| Config/Log 中段写失败 | 记录警告，Finalize 阶段覆盖写兜底 | WARN |
| 日志文件被占用 | 重试机制 + WARN 记录 + 最终状态提示 | WARN |

### 6.3 进度分段模型

**`CreateArchive=true`：**

| 阶段 | 进度区间 | 说明 |
|------|----------|------|
| Init/Prepare/Expand | $0$-$5$ | 初始化与展开 |
| Copy | $5$-$70$ | 文件复制 |
| Save | $70$-$80$ | 写配置副本与首次日志 |
| Archive | $80$-$95$ | 归档 |
| Finalize | $95$-$100$ | 收尾 |

**`CreateArchive=false`：**

| 阶段 | 进度区间 | 说明 |
|------|----------|------|
| Init/Prepare/Expand | $0$-$5$ | 初始化与展开 |
| Copy | $5$-$85$ | 文件复制 |
| Save | $85$-$95$ | 写配置副本与首次日志 |
| Finalize | $95$-$100$ | 收尾 |

### 6.4 安全与一致性

- 仅处理本地文件系统路径
- 执行期锁定 UI，避免并发修改参数
- 结果对象在完成前不对外声明成功
- 日志追加后不可回滚（权威累积语义）

### 6.5 监控与可观测性

- **关键里程碑日志**：Config 加载、APPDIR 解析、Expand 匹配结果、Copy 汇总、Archive 结果、DONE
- **进度文本示例**：`Copying: ...\path\file (12/48)`
- **失败分类**：WARN（可恢复/可跳过）、ERROR（关键失败）
